specs 1.5.0 API

This document is the API specification for specs 1.5.0 API

Class Summary
trait Commentable extends AnyRef
This trait can be added to anything that can be commented.
trait DefaultExecutable extends DefaultResults with Executable with Commentable
This trait represents something that's executable and has results. It can also be commented and not be executed in that case. Subclasses can access the execution status with the "executionStarted" and "executed" variables.
trait DefaultResults extends HasResults
Default implementation for the HasResults trait using lists.
trait Executable extends AnyRef
Anything that's executable and returns itself
case class FailureException (val message : java.lang.String) extends java.lang.RuntimeException with scala.Product
RuntimeException carrying a matcher ko message
trait HasResults extends AnyRef
This trait is useful to get a common interface for anything holding results made of failures, errors and skipped anything, like Specifications, Sus and Examples. This trait is also implemented by the DefaultResults trait using lists to store values. Failures are modeled by FailureExceptions Errors are modeled by Throwables Skipped (meaning "not executed") are modeled by SkippedExceptions
case class SkippedException (val message : java.lang.String) extends java.lang.RuntimeException with scala.Product
RuntimeException carrying a matcher skip message
