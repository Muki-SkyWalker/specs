trait File extends FileSystem with ConsoleLog with SpecsHolder with Console
This trait groups the functionalities of runners creating files from specification executions. It requires the definition of:
  • a file name function
  • an output directory function
  • a specs holder having all the specifications to report
  • a specOutput function creating the output string to copy to the target file
Concrete implementations of that trait can be found in HtmlRunner and XmlRunner.
abstract class FileReporter (outputDirPath : java.lang.String, fName : (BaseSpecification) => java.lang.String) extends File
A FileReporter class is used to have a constructor setting the required variables of the File trait.

A FileSuite is like a FileReporter but supports being extended with the JUnit trait
trait Html extends File
The Html trait outputs the results of a specification execution as an html file in a given output directory. The default file name for the report is "specs-report.html" and that report contains all examples description with their execution status: error, failure, success, skipped.

trait Reporter extends SpecsFilter with ConsoleLog
An object using the reporter trait should be able to report the result of the execution of several specifications. Those specifications are usually provided by the object when using the reportSpecs method, but the report functionality can also be accessed by passing other specifications directly to the report(specs) method. Any object using the Reporter trait will also inherit a main method controlling:
  • the reading of command line arguments
  • the display of stacktraces
  • the acception or rejection of some tags
The accepted arguments are:
  • -ns or --nostacktrace to avoid displaying stacktraces
  • -acc, --accept followed by a comma separated list of tag names for the tags to accept
  • -rej, --reject followed by a comma separated list of tag names for the tags to reject
When subclassing the Reporter trait, the subclasses should usually override the report(specs) method to provide concrete reporting behavior.

Subclasses must not forget to insert a super.report call at the beginning of their processing to allow the chaining of several reporters as traits: object runner extends Runner(spec) with Html with Xml for example
class Runner (val specifications : scala.Seq[Specification], val reporters : scala.Seq[Reporter]) extends Reporter
The Runner class is an abstract class referencing one or several specifications to run. It should be extended with at least one runner trait which will use the specifications. Usage: class mySpecRunner extends Runner(mySpec) with JUnit with Console with ScalaTest or class mySpecRunner extends Runner(mySpec) with JUnit with Xml with ScalaTest which will also output the results in an xml file Then mySpecRunner class can be executed in many ways:
  • scala -cp ... -e "new mySpecRunner.reportSpecs"
  • java -cp ... org.scalatest.Runner -g -s mySpecRunner
  • java -cp ... org.junit.runner.JUnitCore mySpecRunner
It is annotated with a JUnit annotation because JUnit requires that annotation should be placed on the class which will be executed. In the example above, Runner(mySpec) is the only class; JUnit, Console and ScalaTest are all traits.

class SpecsFileRunner (path : java.lang.String, val specFilterPattern : java.lang.String, susFilter : java.lang.String, exampleFilter : java.lang.String) extends SpecsFinder with Console
This class can be used to search for specifications on a given path and execute them.
Usage: object myFileRunner extends SpecsFileRunner(path, pattern)
Where path is a path to a directory containing scala files (it can be a glob: i.e. "dir*spec.scala") and pattern is a regular expression which is supposed to match an object name extending a Specification class named ".*Spec.*" The systems and examples can also be filtered by specifying patterns. These patterns can be passed as System properties, by specifying -Dsus="regexp" or -Dexample="regexp".

trait Xml extends File
The Xml trait is used to create an xml file, in a specified output directory with the results of a specification execution. If the output directory is not specified 
object runner extends XmlRunner(mySpec)
then the current directory will be used Usage: class mySpecRunner extends Runner(mySpec) with Xml The output directory can be overriden if necessary:
 class mySpecRunner extends Runner(mySpec) with Xml { override def outputDir = "./results/specs" }
case class XmlRunner (val specs : scala.Seq[Specification], val outputDirPath : java.lang.String, val fName : (BaseSpecification) => java.lang.String) extends FileReporter with Xml with scala.Product
Concrete class for the Xml trait. It allows to select a specification to run and an output path Usage: object runner extends XmlRunner("./results/specs", mySpec) The name of the generated file is specification.name by default but can be overriden:
 object runner extends XmlRunner("./results/specs", mySpec){ override def fileName="spec-report.xml" }

The XmlSuite is the same class as XmlRunner but can be extended with the JUnit trait.
