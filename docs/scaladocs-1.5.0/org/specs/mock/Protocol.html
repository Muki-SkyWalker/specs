class Protocol
extends ProtocolTypes
The Protocol class stores the expectations of mocks, alongside with the actual received calls It can be exclusive or not: i.e. return an error in case of unexpected calls or not
A protocol has 3 functions:
  • be defined by nested protocol definitions and expected calls
  • store received calls
  • return failures if there are unmatched calls
