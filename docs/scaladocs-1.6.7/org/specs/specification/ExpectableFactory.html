ExpectableFactory

trait ExpectableFactory extends ExampleExpectationsListener with SuccessValues with FailureFactory

This trait defines implicit definitions which are used to create Expectable objects and associate them with the latest defined example
Usage: <code> 2.1 must beCloseTo(2.0, .1) </code>or
<code theDouble(2.1) must beCloseTo(2.0, .1) </code>

Then StringMatchers are expecting values with String as an upper bound so that effectively only String instances will be used with the implicit def (only solution found to make it all work, to my current understanding)
