trait CalledInOrderMatchers
extends ExpectableFactory with NumberOfTimes
This trait provides an additional matcher to check if some methods methods have been called in the right order on mocks: theMethod(m1.get(0)).on(m1).atLeastOnce then theMethod(m2.get(0)).on(m2) were called.inOrder // the implicit definition can also be removed but additional parenthesis are needed (m1.get(0) on m1).atLeastOnce then (m2.get(0) on m2) were called.inOrder
