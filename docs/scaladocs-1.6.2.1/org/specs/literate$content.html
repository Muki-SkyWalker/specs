class LiterateSpecification extends Specification with LiterateBaseSpecification with LiterateSpecificationLinks with LiterateDataTables with LiterateForms with LiterateProperties with LiterateShortcuts
This trait supports writing literate specifications for acceptance testing. The basic idea is to define Systems under specification as some text, enclosed in xml tags: "my system" is some text for my Specification Then, it is possible to "tag" parts of this text with the xml nodes, to denote an example description and make it immediately followed by an expectation in Scala code: "my system" is This is an ok example { 1 must be equalTo(1) } This is an ok example with several expectations which must be enclosed in an 'eg' function { eg { 1 must be equalTo(1) 2 must be equalTo(2) } } This is a example but missing its implementation yet { notImplemented }
trait LiterateSpecificationLinks extends LinkedSpecification with Links
This trait allows to add links to other specifications inside a literate specification. The link will be displayed as a Html link The "parent/child" relationship is kept in that trait to allow the Html runner to be reported when reporting the parent.
