trait Reporter
extends SpecsFilter with ConsoleLog
An object using the reporter trait should be able to report the result of the execution of several specifications. Those specifications are usually provided by the object when using the reportSpecs method, but the report functionality can also be accessed by passing other specifications directly to the report(specs) method. Any object using the Reporter trait will also inherit a main method controlling:
  • the reading of command line arguments
  • the display of stacktraces
  • the acception or rejection of some tags
The accepted arguments are:
  • -ns or --nostacktrace to avoid displaying stacktraces
  • -acc, --accept followed by a comma separated list of tag names for the tags to accept
  • -rej, --reject followed by a comma separated list of tag names for the tags to reject
When subclassing the Reporter trait, the subclasses should usually override the report(specs) method to provide concrete reporting behavior.

Subclasses must not forget to insert a super.report call at the beginning of their processing to allow the chaining of several reporters as traits: object runner extends Runner(spec) with Html with Xml for example
