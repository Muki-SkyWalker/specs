Argument Matchers

Argument matchers

Argument matchers allow flexible verification or stubbing. Click here for more details.

Let's import some definitions first:

import org.specs._
import org.specs.mock.Mockito
import java.util.List
import org.mockito.Matchers.{ argThat, anyInt, eq => isEq }
import org.hamcrest.core.{ IsNull }

And create a specification with mocks:

object args1 extends Specification with Mockito {
val mockedList = mock[List[String]]

// stubbing using built-in anyInt() argument matcher
mockedList.get(anyInt()) returns "element"

// stubbing using hamcrest (let's say IsNull returns your own hamcrest matcher):
mockedList.contains(argThat(new IsNull[String])) returns true
}

Then, calling the mocked list with any argument must return "element":

args1.mockedList.get(999)
> res0: String = element

and calling the mocked list contains method with a valid argument must return "true" if the passed argument is null:

args1.mockedList.contains(null)
> res1: Boolean = true

It is also possible to verify that a mock was called with an argument matcher:

object args2 extends Specification with Mockito {
args1.mockedList.get(999)
args1.mockedList.get(isEq(999)) was called
}
args2.successes
> res2: List[org.specs.specification.Example] = List(example 1)

Instead of Hamcrest matchers, a specs matcher can be used:
object args3 extends Specification with Mockito {
val mockedList = mock[List[String]]
mockedList.get((123)) returns "one"==
}
args3.mockedList.get(123)

> res3: String = one
